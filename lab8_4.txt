4. Purpose of each module
main.c
Entry point of the program. Responsibilities: 1. Read command line arguments 2. Invoke assembler 3. Handle basic errors
Example: main()
 assemble_file("program.s")
assembler.c / assembler.h
This module coordinates the entire assembler pipeline. Responsibilities: open assembly file perform pass 1 perform pass 2 output machine code
This file controls the workflow, but does not implement parsing or encoding itself.
parser.c / parser.h

Responsible for interpreting a line of assembly text. Example input: loop: add x1, x2, x3
Parser produces structured data: label = "loop" mnemonic = "add" operands = ["x1","x2","x3"]
This separation allows parsing to be tested independently.
registers.c / registers.h
Handles register name conversion. Example: "x5"  5 "x31"  31
This module is extremely small but essential.
instr_table.c / instr_table.h
Stores the instruction specification table. Example entry: add format = R opcode = 0x33 funct3 = 0 funct7 = 0

This module demonstrates the key insight: An assembler is largely structured table lookup.
The information comes directly from the CS61C reference card tables.
symbols.c / symbols.h
Stores labels and their addresses. Example symbol table: loop  12 done  24
Responsibilities: insert label lookup label detect duplicate labels
encode.c / encode.h
Responsible for packing fields into machine code. Example: encode_r(opcode, funct3, funct7, rd, rs1, rs2)
Produces a 32-bit instruction.
tests/
Each module has a unit test. Example:

test_registers test_parser test_encode

This reinforces test-driven modular development. In summary, each model solves one problem.

7. Two-pass assembler for labels
Branches reference labels: beq x1 x2 loop
But the address of loop might appear later in the file.
Therefore we need two passes.
Pass 1 -- build symbol table
Read every line and record labels. Example: 0: start: 4: addi x1 x0 5 8: addi x2 x0 10 12: loop: 16: add x3 x1 x2

Symbol table: start  0 loop  12
Pass 2 -- encode instructions
Now branch instructions can compute offsets: offset = label_address - current_PC Example: beq x1 x2 loop Offset = 12 - 4 = 8 This offset is encoded into the B-type immediate field.

Step 6: instruction encoding
Students implement functions: encode_r encode_i encode_s encode_b encode_j

Step 7: implement assembler pipeline
Assembler performs: PASS 1: record labels PASS 2: encode instructions
